多任务

大多数操作系统的基本特性之一就是支持多任务，这是一种允许多任务并行执行的能力。比如，你也许打开了一个程序来看这片文章，比如文字编辑器或者 Terminal 窗口，甚至你可能有一个单独的浏览器窗口开着。多重多样的后台任务在运行，为了你可以管理桌面窗口，检查更新或者索引文件。
任务看起来是并行的，但 CPU 只允许一个任务在执行。为了创建同时运行的错觉，操作系统迅速的切换在活动的任务中。
单核 CPU 同时只能运行一个任务，但多核 CPU 可以真正同时运行多个任务。比如8core CPU 可以同时运行8个任务。我们会解释如何装载多核 CPU。
有两种形式的多任务系统：合作式多任务需要任务主动放弃 CPU 的控制权，以便其他任务可以进行。抢占式多任务使用操作系统提供的能力去切换线程，操作系统可以任意的强制停止他们。在接下来，我们将解释两种不通形式。

抢占式多任务

抢占式多任务的核心关键是由操作系统来控制何时切换任务。正因如此，每个 Interrupt 都会重新获得控制权。这也使得无论何时系统中有新的输入可用时得以切换任务。比如，鼠标移动或者网络包到达。操作系统可以确定确切的时间一个任务可以运行通过配置一个硬件时间，发送一个信号在此以后。
